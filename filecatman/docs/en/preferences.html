Preferences

The preferences dialog allows the user to customize application settings as well as database specific settings.

General

Here you can set interface settings, such as the icon theme and GUI style.

Icon Theme

Sets the icon theme to the one specified.

Style

Set the GUI style. Styles may be different on each platform.

Note

Changing this setting requires an application restart.

Use style's standard palette

If checked, the selected GUI style's color scheme will be used, else the color scheme from the default style will be chosen.

Note

Changing this setting requires an application restart.

Database

Database specific settings can be changed here. Such as the project's data directory, and whether to open the database on startup.

Open current database on startup.

Clicking this checkbox will enable the option to automatically open the database at startup, skipping the startup wizard.

Data Directory

This is the folder that holds the data files for your project. The files correspond to the items that are in the database.

Category Descendant Levels

The maximum number of levels from a root category that a child category is allowed to descend.

Note

Increasing the level will make it take longer to load the category hierarchy.

Item Types

This tab displays the item types that the current database supports. Options available in a buttonbox below the table, allows the user to add, edit, delete and rearrange item types.

Visible

If unchecked, the selected item type will be hidden from view in the main window.

Add

Opens the “New Item Type Dialog”. An item type can then be created by filling the necessary details, such as it's name, and which file extensions it supports.

Edit

Opens the “Edit Item Type Dialog”.

Delete

Delete the selected item type. All items with this type will be deleted. A confirmation dialog will be displayed, with the option to delete files in the data directory with the items.

Warning

The deletion of items and files cannot be undone.

Move Up List, Move Down List

Rearrange the item types in a certain order.

Add Defaults

Any default item types that have been deleted from the list, will be added again.

Reset

Any current changes that have been made to the item types in the preferences dialog will be reverted.

New Item Type

In this form, details for a new item type can be given. An item type has different names for different purposes. The extensions tab is where a list of supported file extensions can be specified.

Icon

Select an icon for this item type. Clicking the icon opens the “Pick Icon Dialog”, where the user can choose an existing icon, or upload a new icon.

Noun Name

The noun of the item type name.

Plural Name

The plural of the item type name.

Table Name

The name for the item type in the database.

Folder Name

The name of the data directory folder for this item type, as well as the folder name used when creating symbolic links.

Visible

If unchecked, this item type will be hidden from view in the main window.

Extensions

In this tab the user can specify which file extensions the item type relates to.

Tip

Leave the extension list blank, and the item type will become a weblink. This is useful, if the user wants multiple weblink types.

Tip

If the extensions list contains both html and htm, then the item type will become a webpage.

Caution

An extension must not be used in more than one item type.

Taxonomies

This tab displays the taxonomies that the current database supports. Options available in a buttonbox below the table, allows the user to add, edit, delete and rearrange taxonomies.

Visible

If unchecked, the selected taxonomy will be hidden from view in the main window.

Add

Opens the “New Taxonomy Dialog”. A taxonomy can then be created by filling the necessary details, such as it's name and whether it will be in tags mode.

Edit

Opens the “Edit Taxonomy Dialog”.

Delete

Delete the selected taxonomy. All categories with this taxonomy will be deleted. A confirmation dialog will be displayed.

Warning

The deletion of categories cannot be undone.

Move Up List, Move Down List

Rearrange the taxonomies in a certain order.

Add Defaults

Any default taxonomies that have been deleted from the list, will be added again.

Reset

Any current changes that have been made to the taxonomies in the preferences dialog will be reverted.

New Taxonomy

In this form, details for a new taxonomy can be given. A taxonomy has different names for different purposes. A taxonomy can be specified to be displayed in tags mode, and can have child categories disabled.

Icon

Select an icon for this taxonomy. Clicking the icon opens the “Pick Icon Dialog”, where the user can choose an existing icon, or upload a new icon.

Noun Name

The noun of the taxonomy name.

Plural Name

The plural of the taxonomy name.

Table Name

The name for the taxonomy in the database.

Folder Name

The folder name used when creating symbolic links.

Visible

If unchecked, this taxonomy will be hidden from view in the main window.

Has Children

If unchecked, categories in this taxonomy cannot have children.

Tags Mode

If checked, the categories will be displayed as a tags list in the “New/Edit Item Dialog”.

Copypasta

Settings for the Copypasta feature can be changed here. The fields to be copied can be enabled or disabled, and the formatting tab allows custom formatting to be applied to the copypasta.

Name

The name field of the items to be copied.

Source

The source field of the items to be copied.

Description

The description field of the items to be copied.

Don't remove file extension from name.

Choose to keep the file extension in the name field.

Formatting

This tab is where advanced custom formatting settings can be chosen. When the formatting settings have been applied, copypasta will be styled with this formatting. The area where the field text will be added to the formatting markup is represented with a {0} command. Multiple of these commands can be added to a single field, this is useful for when a formatting markup needs the text to be specified twice, such as an html link. Formatting settings can be saved to templates and selected when needed.

Field

The name of the item's field.

Formatting

The formatting markup to be applied to the field.

Outer Formatting

This is the formatting which will wrap the copypasta for each item.

Format each line of description.

When checked, the formatting markup will be applied to each sentence in the description, rather than the entire description.

Edit

Edit the selected field's formatting.

Clear

Sets the selected field's formatting back to {0}.

Reset

Any changes that have been made to the current formatting markup will be reverted.

Templates

The formatting markup can be saved to a template, so markups can easily be switched when needed.
